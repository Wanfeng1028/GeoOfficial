## 问题

分析某城市过去若干年建设用地扩张范围、速率与主要转化来源，回答两个问题：

- 哪些区域的建成区在扩张？
- 新增建成区主要由哪些地类转化而来？

## 数据

所有数据来自公开档案，可在 GeoWork 中通过 GDAL 或 GEE 直接读取：

- **Landsat 5 / 7 / 8 Collection 2 Level-2**：USGS Earth Explorer 公开存档，30 m 多光谱，16 天重访。来源：`https://earthexplorer.usgs.gov/`
- **Sentinel-2 L2A**：Copernicus Open Access Hub，10 m / 20 m 分辨率，5 天重访。来源：`https://scihub.copernicus.eu/`
- **行政边界**：自然资源部公开数据或 GADM 行政边界数据集（v4.1）。来源：`https://gadm.org/`
- **土地覆盖参考**：ESA WorldCover 2021（10 m 分辨率）。来源：`https://esa-worldcover.org/`

## 工具

- **GEE**：影像合成、云掩膜、计算 NDBI 与 NDVI
- **GDAL**：本地栅格裁剪、投影变换、格式转换
- **QGIS**：可视化、人工目视核对、制图输出
- **Python（Geo Worker）**：变化检测、面积统计、生成图表
- **Skills**：将"多时相变化检测"封装为可复用工作流

## 流程

1. **项目初始化**：在 GeoWork 中创建 `urban-expansion` 项目，导入研究区矢量边界与时间窗口（例如 2015–2024）
2. **数据准备**：通过 GEE Skill 拉取 Landsat 与 Sentinel 影像集合，按年度合成无云影像
3. **特征计算**：计算 NDBI（建成区指数）与 NDVI（植被指数），生成年度特征栈
4. **变化检测**：使用阈值法或随机森林分类，识别新增建成区像素
5. **统计**：按行政边界汇总新增建成区面积，并按 ESA WorldCover 类别统计转化来源
6. **输出**：生成变化图、面积统计表、方法记录 Markdown 报告

## 验证

- **精度评估**：在每个时相随机抽取 100 个验证点，目视解译后计算混淆矩阵与 Kappa 系数
- **交叉验证**：与 ESA WorldCover 2021 的建成区图层做空间叠加，差异像素需 < 15%
- **敏感性检查**：对 NDBI 阈值 ±0.05 做敏感性测试，确认结果稳定性
- **可追溯性**：所有处理步骤记录在 Project Context 的 Task 历史中，参数可回溯

## 成果

实际输出（示例，具体数值取决于研究区与时间窗口）：

- **年度变化图**：2015 → 2024 共 10 张建成区掩膜
- **面积统计表**：按行政区汇总的新增建成区面积（公顷）
- **转化矩阵**：新增建成区来源地类（耕地 / 林地 / 草地 / 水体）
- **方法报告**：包含参数、数据源、精度评估的 Markdown 文档

> 所有数字必须来自上述公开数据源或用户实际运行结果，禁止编造。

## 限制

- 影像精度受云覆盖与时间分辨率限制
- 阈值法在城乡接合部可能出现误分类，需人工核对
- 当前自动化能力属于 Developer Preview，部分步骤仍需手动确认
