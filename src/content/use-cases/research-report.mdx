## 问题

整理研究文献、复现方法步骤、生成可检查的实验报告。具体回答：

- 如何将分散的文献、代码、数据组织成一份可追溯的科研报告？
- 怎么让方法的每一步都可被同行复核？

## 数据

所有数据来源必须明确标注：

- **文献库**：用户自行导入的 PDF（来自 IEEE Xplore / Web of Science / arXiv 等公开数据库）
- **实验数据**：用户上传的原始数据（栅格 / 矢量 / CSV），需自带来源说明
- **代码与笔记**：用户已有的 Python 脚本、Jupyter Notebook 或 GeoWork Skills
- **参考实现**：可选引用 GitHub 开源仓库（来源：`https://github.com/`）

## 工具

- **Python（Geo Worker）**：复现方法步骤、生成图表、统计计算
- **Skills**：将"文献综述 → 实验复现 → 报告生成"封装为可复用工作流
- **MCP**：让 AI Agent 协助整理文献摘要、生成图表说明
- **Markdown / PDF 导出**：最终报告输出格式
- **Reporting**：生成图表、统计与方法报告

## 流程

1. **项目初始化**：在 GeoWork 中创建 `research-report` 项目，导入文献 PDF 与实验数据
2. **文献整理**：通过 MCP 让 AI 提取每篇文献的方法、数据、结论，写入 Project Notes
3. **方法记录**：在 Code 模式中记录实验步骤与参数，每一步保存为 Task
4. **复现执行**：运行 Python 脚本或 Skills，生成图表（matplotlib / plotly）与统计结果
5. **报告草稿**：将文献综述、方法、结果、讨论拼接为 Markdown 草稿
6. **导出**：渲染为 PDF 或保留 Markdown，所有图表与代码均可点击追溯

## 验证

- **可追溯性**：报告中每个图表必须关联到具体的 Task ID 与代码片段
- **参数一致性**：报告中的参数与代码中实际运行的参数必须一致（由 Project Context 自动校验）
- **引用核对**：所有文献引用需在文末参考文献列表中对应，DOI 链接可点
- **方法复核**：同行可通过 Project Context 中的 Task 历史逐步复核每一步

## 成果

实际输出（示例，具体内容取决于研究主题）：

- **实验报告**：包含摘要、文献综述、方法、结果、讨论、参考文献的 Markdown / PDF
- **图表集**：matplotlib / plotly 生成的图表，每张关联到 Task ID
- **方法记录**：所有参数、代码片段、运行日志的归档
- **参考文献**：DOI 链接列表，可点击追溯

> 所有数字与结论必须来自用户实际运行结果或所引用文献，禁止编造。

## 限制

- 文献 PDF 内容需用户自行确认版权与使用许可
- AI 自动生成的文献摘要属于辅助能力，结果需人工核对
- 当前 Markdown → PDF 渲染对中文支持依赖系统字体配置
